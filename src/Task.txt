Написать rest api aka твиттер
Есть пользователи: user, moderator
Юзер может: 
	+1)писать твиты(ограничение 200 символов на твит), 
	+2)писать треды(неограниченное количество твитов, последовательно связанные между собой)
	3)ставить лайки(только на чужие твиты),
	4)делать репосты с комментариями и простые репосты без комментариев (и своих и чужих твитов),
	5)удалять свои твиты (при удалении твита все его репосты удаляются,если удален твит из треда,то сам тред остается)
	+6)подписываться на других юзеров
Моедратор может:
	1)удалять твиты
	+2)удалять аккаунты
	+3)создавать других модераторов
Секъюрити через JWT,время жизни токена - 1 сутки,автоматический реврешом токена, если юзер не выходил из аккаунта,
когда юзер выходит из аккаунта токен становится невалидным
-/+ Текст твиттов храним в mongoDb,все остальное в постгре
Api может:
	1)получение всех твитов какого-то юзера
	2)получение конкретного твита
	3)получение репостов конкретного твита 
	+4)поиск юзеров по никнейму,в том числе поиск по части никнейма
	5)получение треда по конкретному твиту
	6)получение ленты - список всех твитов пользователей,на которых подписан юзер,в сортировке от самых новых до самых старых(только для авторизированных)
	7)репост(только для авторизированных)
	8)лайк(только для авторизированных)
	9)регистрация юзера по уникальному email,никнейм тоже должен быть уникальным(может включать в себя цифры)(только для не авторизированных)
	+10)удаление твита(только для авторизированных)
	+11)создание твита(только для авторизированных)
	+12)удаление собственного аккаунта(только для авторизированных)
	+13)восстановление аккаунта,если после удаления прошло меньше 6 месяцев
	14)создание нового модератора(только для модератора)
	15)удаление твита какого-то пользователя(только для модератора)
	16)удаление аккаунта какого-то пользователя(только для модератора)
Апи должно поддерживать пагинацю
